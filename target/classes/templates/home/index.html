Información de la web

Utilizacion de la pagina

Esta pagina utiliza en su backend el conocido framework de Spring Batch. Este framework se va a encargar de mostrar los datos de un CSV importado e insertarlos en la base de datos. Estos datos no van a poder ser editados, es decir, tienen que ser unos datos especificos, ya que las limitaciones de codigo no dejan tener varios CSVs con diferentes tipos de datos.

Sin embargo, para que el usuario pueda comprobar el funcionamiento de Spring Batch se va a proprcionar una pagina la cual va a descargar un CSV usable. Esta pagina se encuentra disponible en todo momento en la esquina superior derecha con el boton verde de CSV.

Tambien se podra acceder a ella desde la pagina de añadir usuarios, la cual se usa para esto mismo, importar usuarios desde un CSV, que sean leidos, procesados y escritos en la base de datos.

Base de Datos

La base de datos de este pryecto al igual que la web, pertenece a un servido, por lo cual es accesible desde cualquier maquina sin necesidad de transportar la db junto con el proyecto. Sin embargo, para comprobar el funcionamiento de la misma, al igual que los datos importados en ella, requiere de ciertos pasos a seguir.

CSV y Servidor

Aunque ya ha sido explicado en la primera pestaña, el CSV proporcionado aqui, pueden ser cambiados los valores. Sin embargo, el cambio de estos va a alterar al funcionamiento de Spring Batch, pueden ser cambiados los valores numericos, fechas, etc. Todo puede ser cambiado mientras no se modifique la estructura de los datos, ni el nombre de los mismos (ya que son usados en las columnas de la base de datos), ni el tipo de datos (strings, doubles, etc).

El servidor que aloja a la web (el mismo que aloja a la base de datos) es Render. Este servidor es gratuito, por lo que tiene ciertas limitaciones. Sin embargo es de los pocos que admiten tanto backend como frontend a la vez, por lo que es el que se ha elegido.

El link anterior propociona la "dashboard" del servidor, donde se pueden ver los datos de la web, capacidad de procesamiento, etc. Tambien se puede acceder a la seccion más importante de todas, los "Deployments". Estos son los que siguen un registro de como ha sido creada la web (a parte de Github), y se puede acceder a ellos para ver los logs de la web, los errores, etc. Estos deployments tienen su propia consola la cual enseña como esta creandoase y funcionando la web en tiempo real.Tambien se puede ver como es la introduccion de los datos de un CSV, ya que Spring Batch muestra como esta siendo el proceso de la impirtacion de datos del CSV a la db.
